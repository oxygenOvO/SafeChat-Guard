# 项目整体框架

## 1. 比赛题目定位

题目 1 的本质是工程实现题，不是从零训练大模型。核心评分点是：功能完整、过滤效果、代码质量、文档质量。

因此本项目要优先保证系统跑通，再逐步加入大创项目里的创新点。

## 2. 最终系统要做出来什么

一个 Web 系统，用户输入一句话后，系统完成：

1. 输入文本归一化：处理谐音、表情、变体写法。
2. 第一层过滤：关键词词库和正则规则快速匹配。
3. 第二层过滤：文本分类模型判断疑似内容。
4. 分级处理：高风险拦截，低风险脱敏，正常放行。
5. 调用大模型：将安全输入转发给轻量大模型接口。
6. 输出校验：对模型回复再过滤一次。
7. 日志记录：保存输入、输出、命中规则、处理结果。
8. 统计展示：展示拦截次数、脱敏次数、风险类别分布。

## 3. 与 SIT 大创项目的相关性

| 比赛模块 | 大创项目对应点 | 作用 |
| --- | --- | --- |
| 中文违规内容识别 | 中文毒性文本缓解 | 作为输入/输出安全检测基础 |
| 谐音、表情、变体处理 | 中文对抗样本增强 | 作为中文场景创新点 |
| 二次语义分类 | XLM-R / RoBERTa 跨语言迁移 | 替代简单关键词误判 |
| 低风险改写 | LLM detox / ParaDetox 思路 | 实现“拦截之外的修正” |
| 价值观评分 | 价值对齐评估 | 作为可视化亮点和报告创新点 |

## 4. 推荐四人分工

| 角色 | 负责内容 | 第一周交付 |
| --- | --- | --- |
| A 组：规则与词库 | 四类词库、正则、归一化映射、规则测试 | `data/lexicons` 和 `data/rules` 初版 |
| B 组：语义分类与分级 | 二次分类模型、风险分数、分级策略 | `semantic_classifier.py` 可替换真实模型 |
| C 组：后端与接口 | 后端、pipeline、日志、API | 系统可以端到端运行 |
| D 组：前端与文档 | Web 页面、统计面板、报告、截图 | 演示界面和文档框架 |

## 5. 技术路线

```text
输入文本
  ↓
归一化 normalizer
  ↓
关键词 + 正则 rule_filter
  ↓
语义分类 semantic_classifier
  ↓
风险分级 pipeline
  ↓
拦截 / 脱敏 / 放行
  ↓
LLM 回复
  ↓
输出再次过滤
  ↓
日志与统计
```
